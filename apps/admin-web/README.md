# Admin

The admin panel is the command center for Starkeep. Start here — it guides you through the full setup process and provides ongoing visibility into local and cloud state.

## Running

```bash
pnpm --filter admin-web dev
```

Opens on port 3000.

## What It Does

### Dashboard

The dashboard shows the current state of the local and cloud environments:

- **Local status** — whether the data-server is running, which record types are registered, file watch status, and record counts per type
- **Cloud status** — whether cloud infrastructure is provisioned and reachable
- **App status** — whether photos-web and file-browser are running locally

### Cloud Setup Wizard

A step-by-step wizard that walks you through provisioning cloud infrastructure for a new user:

1. Enter bootstrap stack outputs (the CloudFormation outputs from the control plane stack)
2. Create a Cognito user account
3. Sign in to verify credentials
4. Deploy the IAM permissions stack
5. Provision the user's full infrastructure (Aurora DSQL, S3, API Gateway, Lambda)

The wizard writes a `starkeep-config.json` file that the data-server reads to connect to the cloud.

### Settings

- **Import/export cloud config** — move configuration between machines or back it up
- **Redeploy infrastructure** — trigger a fresh deployment of the user's cloud stack
- **Reset wizard** — clear config and start the setup process over

### Permissions

Manage the IAM permission stacks that authorize Starkeep to provision and access AWS resources. You can view the IAM statements grouped by component (SST, Aurora DSQL, S3, Lambda, API Gateway, CodeBuild), and create, update, or delete permissions stacks via CloudFormation.

## Configuration

The admin panel reads cloud configuration from `starkeep-config.json`. This file can be:

- Generated by the cloud setup wizard
- Imported manually via the Settings page
- Located at the repo root or pointed to via the `STARKEEP_CONFIG` environment variable

Cognito credentials (for signing in to the user's AWS Cognito pool) are stored in browser localStorage and used to authenticate requests to the data-server and cloud API.
