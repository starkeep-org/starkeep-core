# Tasks App

The Tasks app is a task management tool built on the Starkeep protocol. It ships as two
separate applications — a web app (Next.js) and a desktop app (Tauri) — that share the
same data model, domain logic, and UI components. Both can sync to the same cloud
infrastructure.

The app serves as the reference implementation for building on Starkeep. See
[Building an App](building-an-app.md) for a walkthrough of the patterns it uses.

## Core concepts

**Groups** are the primary container. Every task belongs to exactly one group. Users can
own groups and can join other users' groups as collaborators. A new user starts with a
Personal group created automatically.

**Tasks** are the unit of work within a group. Each task has structured fields (title,
status, assignee, labels) plus a comment thread and a change history.

**Views** are saved filters and sort orders for a group. A view defines which tasks to
show and in what order, and can be scoped to a specific assignee, set of statuses, or
labels. Views are named and persist between sessions.

**Importance ordering** is a user-defined ranking of tasks within a group by priority —
independent of status. Tasks higher in the importance order are considered more important.

## Task fields

| Field | Description |
|-------|-------------|
| **Title** | Short summary of the task |
| **Description** | Detailed notes |
| **Status** | Current state in the task lifecycle (see below) |
| **Assignee** | Name of the person responsible |
| **Labels** | Free-form tags; multiple allowed |
| **Blockers** | Dependencies preventing progress — either links to other tasks by ID or free-text descriptions of external blockers |
| **Comments** | Threaded discussion; each comment has an author, content, and timestamp |

### Status lifecycle

```
Blocked ──> Backlog ──> Todo ──> In Progress ──> Done
```

Tasks begin in Backlog by default. A task in Blocked is waiting on another task or
an external dependency to be resolved before it can be worked on.

## Group management

- **Create a group** — give it a name and optional description
- **Rename or update** a group's name or description
- **Delete a group** — removes the group and all its tasks
- **Share a group** — generate an invite token that grants a collaborator read (or
  read/write) access
- **Join a group** — enter an invite token to add a hosted group to your sidebar

Hosted groups (owned by others) appear alongside your own groups and sync
independently.

## Task management

- **Quick-add** — type a title in the input at the bottom of the task list and press Enter
- **Full form** — open a task to edit all fields: title, description, status, assignee,
  labels, and blockers
- **Delete** — remove a task from the group permanently
- **History** — every change to a task is recorded; view the full audit trail from the
  task detail panel

## Search and filtering

**Full-text search** matches task titles and descriptions across all tasks in a group.

**Views** allow you to define persistent, named filters:

| Filter | Description |
|--------|-------------|
| Status filter | Show tasks with specific statuses (e.g., only Todo and In Progress) |
| Assignee filter | Show tasks assigned to a person; `{{currentUser}}` resolves to the signed-in user |
| Label filter | Show tasks with specific labels |

Each view also has an ordering mode:

| Mode | Description |
|------|-------------|
| **Importance order** | User-defined priority ranking within the group |
| **Comprehensive order** | Ordering that factors in status, priority, and other signals |

## Task ordering

Within a group, tasks have an **importance ordering** — a ranked list from highest to
lowest priority. Reorder tasks by dragging them in the task list when importance order
mode is active.

**AI-assisted placement**: when creating a new task, the app can suggest where to insert
it in the importance ordering. The suggestion is generated by Claude based on the task's
title, description, and the existing tasks in the group.

## Collaboration

- **Sync** — changes made on one device appear on others after sync. The desktop app
  syncs automatically in the background; the web app syncs on load and on demand.
- **Comments** — add comments to any task; comments show the author's name and timestamp
- **Group sharing** — share a group via an invite token; revoke access by removing the
  policy

## UI layout

The app uses a three-column layout:

```
+------------------+------------------+----------------------+
|  Chat            |  Task List       |  Task Detail         |
|                  |                  |                       |
|  Conversational  |  Filtered list   |  Title, description,  |
|  interface       |  for active view  |  status, assignee,   |
|                  |                  |  labels, blockers,    |
|  (side panel)    |  [+ quick-add]   |  comments, history    |
+------------------+------------------+----------------------+
```

- **Chat panel** — a conversational AI interface for interacting with your tasks
- **Task list panel** — shows tasks filtered by the active view; the quick-add box
  is at the bottom
- **Task detail panel** — opens when a task is selected; shows the full task form,
  comment thread, and change history

The view picker appears above the task list and lets you switch between saved views
with a tab-like interface.

## Settings

| Setting | Description |
|---------|-------------|
| Display name | Your name as it appears in comments and task assignments |
| Theme | Light, dark, or system (follows OS preference) |
| Auto-sync | Enable or disable automatic background sync |
